# ISSUE-08：主控台範例日期去硬編碼

**Labels:** `priority:low` `type:chore` `area:skill-主控台`
**Status:** open

## Context

診斷初稿曾把主控台 §學習與生活 的「沖繩 6/4–6/8」當成過期的系統狀態（B2）。

**Sean 更正**：那是**範例**（原文「近期家庭行程（如沖繩 6/4–6/8）」），**不是系統邏輯問題**。診斷初稿判錯了嚴重度。

## Problem（降級後的真實問題）

範例用了**具體的過去日期**，讀起來會像「過時的真實狀態」，造成誤讀。純措辭問題，非邏輯缺陷。

## Proposal

範例改為不帶死日期的通用描述，或明確標〔範例〕。例如：
> 🎓 學習與生活：PMBA 關卡、近期家庭行程（例：旅遊／回診／學校活動）

## Acceptance Criteria

- [ ] 範例不含具體可過期的日期
- [ ] 仍清楚傳達該區塊要放什麼
