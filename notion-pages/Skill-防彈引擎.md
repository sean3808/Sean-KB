# 🛡 防彈引擎 ｜ 操作技能

> 🛡 **一句話定位**：把防彈筆記法（Esor，2022）的操作心法，變成蒜頭每次互動時實際「用出來」的引擎——讓每則筆記都為了推進未來行動而存在。

## 觸發詞
`防彈`、`防彈引擎`、`防彈檢查`、`用防彈拆一下`
（偵測到 → 讀本頁 → 依下方流程執行）

## 核心心法（11 條）
1. **一任務一筆記**：把散落在 email／即時通／會議的碎片，整合進「以任務為單位」的單則筆記。（A001–A003、A048）
2. **產出才是任務單位**：任務的單位是「可交付的具體產出」，不是資料、也不是行動清單。（A021）
3. **成果三問**：為誰為何而做？怎麼算完成（可檢查成果）？有何阻礙限制？（A007）
4. **時間管理＝設計任務與行動**，不是排時間；做不到先想「更有動力的成果／更有效的承諾／做得到的行動」。（A041）
5. **三種誘因**讓計畫真的被執行：成果誘因、約定誘因、壓力誘因。（A037–A040）
6. **收集箱是緩衝層**：雜事先進收集箱，下一個空檔再判斷歸屬，不直接塞進待辦。（A046–A047）
7. **KPT 覆盤**：Keep／Problem／Try，把覆盤變成下次可用的行動，而非寫日記。（A014）
8. **筆記升級鏈**：任務筆記 → 永久任務筆記 → 專案目錄 →（結案）有結構封存。（A015、A058）
9. **掌控全局四系統**：整理／選擇／提醒／回顧，缺一不可。（A063）
10. **習慣是實驗非立志**；要改變優先「改工作流程」而非「加額外習慣」。（A066、A077）
11. **避免過度整理**：專注紀錄而非設計功能；整理得再好，不如寫下兩步行動。（A010、A018、A008）

## 觸發後的操作流程（防彈檢查）
針對「當前頁面／任務」依序快檢並輸出：
- [ ] 這頁角色清楚嗎？（一專案一核心頁；會議／子任務用連結納入）
- [ ] 有沒有可檢查的**完成定義**？沒有就用成果三問補。
- [ ] 下一步行動是否**具體到一看就懂**、且**只放我方可推進**？
- [ ] 有沒有指向**產出**（而非只是「找資料」這種無成果行動）？
- [ ] 阻礙／外部依賴 → 對應「我方最小跟進動作」。
- [ ] 要不要**升級或封存**（見下方升級路徑）？

## 升級路徑（對應 Sean 的 Notion）
| 防彈階段 | Sean 的實作 |
|---|---|
| 收集箱（暫存碎片） | 🧩 碎片日誌 |
| 核心任務筆記 | 工作紀錄庫 → 工作交辦頁 |
| 永久任務筆記／專案目錄 | 🗂 專案管理頁 |
| 有結構封存（經驗沉澱） | 「另存回母專案」／achieve old data |

碎片勾「升級」→ 週回顧時協助搬移成獨立任務頁；任務結案 → 提醒把可複用流程沉澱回母專案。

## 防彈四等級（自評用）
1. 直接行動但過程混亂
2. 可確認成果輕重緩急（已能形成核心任務筆記）
3. 可提前發現目標與流程問題（看見風險）
4. 可採取有價值行動（完成有價值任務，而非整理資料）

目標：把 Sean 的任務穩定維持在第 2～3 級以上。

## 三種誘因（落地話術）
- **成果誘因**：把「找資料」改寫成「找到 3 個案例」這種當下可交付的小成果。（A038）
- **約定誘因**：對他人做承諾（寄信、約會議、約定回報）才持久。（A039）
- **壓力誘因**：先限時、再拆步驟，產生「知道方向的緊迫感」。（A040）
